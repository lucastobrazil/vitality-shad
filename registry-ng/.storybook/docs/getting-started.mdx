# Vitality UI for Angular

Vitality is a custom component registry built on top of [shadcn/ui](https://ui.shadcn.com), ported to Angular with Tailwind CSS v4.

## Installation

### 1. Set up `shadcn-ng` for Angular in your Angular project

Ensure [Tailwind CSS v4](https://tailwindcss.com/docs/installation/framework-guides/angular) is configured and working in your Angular app. Then initialize the registry:

```bash
npx shadcn-ng@latest init
```

This CLI has interactive prompts that are difficult to automate non-interactively. The init process:

1. Asks for theme (selected: neutral)
2. Asks for app config file path (selected: `src/app/app.config.ts`)
3. Asks for global CSS file (selected: `src/styles.css`)
4. Asks for component alias (selected: `@/shared/components`)
5. Creates `components.json` at the project root

**Gotcha:** The init CLI did NOT write the CSS theme to `styles.css` despite completing successfully. The theme was manually copied from the `shadcn-ng` package's theme-definitions file at `node_modules/shadcn-ng/dist/core/themes/theme-definitions.js`.

The full theme is defined in `src/styles.css` using:

- `@theme inline { ... }` block for Tailwind v4 theme tokens (maps CSS custom properties to Tailwind utility names)
- `:root { ... }` for light mode CSS custom property values
- `.dark { ... }` for dark mode overrides
- `@custom-variant dark (&:is(.dark *))` for class-based dark mode

Color tokens use oklch color space. The theme includes: primary, secondary, muted, accent, destructive, success, warning, info, brand, card, popover, sidebar, header, footer, code, chart-1 through chart-5, and border/input/ring.

Semantic font size and line height tokens are also defined (caption, body, button, textInput, sectionSubtitle, sectionTitle, pageTitle).

Radius uses a custom system mapping Tailwind's radius scale to `--radius-default`, `--radius-large`, and `--radius-rounded` variables.

### 3. Add components from the Vitality registry

Point the CLI at the Vitality registry URL when adding components:

```bash
npx @ngzard/ui@latest add https://vitality-shad.vercel.app/r/ng/<component>.json
```

For example, to install the Button component:

```bash
npx @ngzard/ui@latest add https://vitality-shad.vercel.app/r/ng/button.json
```

### 3. Import and use

All components are standalone. Import them directly in your component or module:

```typescript
import { ZardButtonComponent } from "@/ui/button/button.component";

@Component({
  imports: [ZardButtonComponent],
  template: `<button z-button zVariant="primary">Click me</button>`,
})
export class MyComponent {}
```

## Extra Setup

### Path aliases

Components use `@/ui/` and `@/lib/` path aliases. Add these to your `tsconfig.json`:

```json
{
  "compilerOptions": {
    "paths": {
      "@/ui/*": ["./vitality/ui/*"],
      "@/lib/*": ["./vitality/lib/*"]
    }
  }
}
```

## Overview of Stack

- **Framework:** Angular 21.1.4 (standalone components, signals)
- **CSS:** Tailwind CSS v4.1.18 via `@tailwindcss/postcss`
- **Component library:** zardUI (`@ngzard/ui`) â€” a shadcn/ui-equivalent registry for Angular
- **Custom registry:** Components sourced from `https://lucastobrazil.github.io/vitality-shad/r/ng/`
- **Variant system:** class-variance-authority (CVA)
- **Utility merging:** clsx + tailwind-merge
- **Icons:** lucide-angular
